## 一、基本的 Git 命令及其用法如下（附带示例）：

### 1. `git status`

查看当前仓库的状态，包括有哪些文件被修改或未暂存。

**用法：**

bash

```
git status
```

**输出示例：**

代码

```
On branch main
Your branch is up to date with 'origin/main'.

Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
    modified:   file1.txt
```

------

### 2. `git init`

在当前目录初始化一个新的 Git 仓库。

**用法：**

bash

```
git init
```

**输出示例：**

代码

```
Initialized empty Git repository in /path/to/your/project/.git/
```

------

### 3. `git clone`

从远程仓库克隆一个项目到本地。

**用法：**

bash

```
git clone <仓库地址>
```

**例子：**

bash

```
git clone https://github.com/user/repo.git
```

------

### 4. `git add`

添加文件到暂存区，准备提交。

**用法：**

bash

```
git add <文件名>
# 或添加所有文件
git add .
```

**例子：**

bash

```
git add main.py
git add .
```

------

### 5. `git commit`

将暂存区的文件提交到本地仓库，并写一条提交说明。

**用法：**

bash

```
git commit -m "提交说明"
```

**例子：**

bash

```
git commit -m "修复了bug，增加了新功能"
```

------

### 6. `git push`

将本地仓库的提交推送到远程仓库。

**用法：**

bash

```
git push <远程名> <分支名>
```

**例子：**

bash

```
git push origin main
```

------

### 7. `git pull`

从远程仓库拉取并合并最新的更改到本地分支。

**用法：**

bash

```
git pull <远程名> <分支名>
```

**例子：**

bash

```
git pull origin main
```

------

**总结：**

- `git status`：查看当前状态
- `git init`：初始化本地仓库
- `git clone`：克隆远程仓库
- `git add`：添加到暂存区
- `git commit`：提交到本地
- `git push`：推送到远程
- `git pull`：拉取远程更新

这些是日常用 Git 管理代码的基础命令，熟练掌握后可以高效协作开发。

## 二、Git创建和管理分支

### 1. 创建分支

分支允许你在不影响主分支的情况下开发新功能或修复 bug。

**命令：**
```sh
git branch <分支名>
```

**示例：**
```sh
git branch new-feature
```
这会创建一个名为 `new-feature` 的新分支，但不会自动切换到该分支。

---

### 2. 切换分支

你可以用 `git checkout` 或新的 `git switch` 命令在分支之间切换。

**命令：**
```sh
git checkout <分支名>
# 或（推荐使用）
git switch <分支名>
```

**示例：**
```sh
git switch new-feature
```
这会切换到 `new-feature` 分支。

---

### 3. 创建并切换分支（一步完成）

你可以用一条命令同时创建并切换到新分支：

```sh
git checkout -b <分支名>
# 或（推荐）
git switch -c <分支名>
```

**示例：**
```sh
git switch -c experiment-branch
```

---

### 4. 合并分支

当分支上的开发完成后，通常会把它合并回主分支（比如 `main`）。

**命令：**
```sh
git checkout main           # 切换到主分支
git merge <分支名>          # 合并指定分支
```

**示例：**
```sh
git checkout main
git merge new-feature
```

---

### 5. 变基分支（rebase）

变基可以让你的分支在目标分支的最新提交上重新应用提交，获得更直线的提交历史。

**命令：**
```sh
git checkout <你的分支>
git rebase <目标分支>
```

**示例：**
```sh
git checkout new-feature
git rebase main
```

这样会把 `new-feature` 分支的提交“移动”到 `main` 分支的最新提交之后。

---

### 6. 删除分支

合并完成后，可以删除已无用的分支：

**命令：**
```sh
git branch -d <分支名>        # 分支合并后安全删除
git branch -D <分支名>        # 强制删除，不管是否已合并
```

**示例：**
```sh
git branch -d new-feature
```

---

### 7. 查看分支

**命令：**
```sh
git branch
```
可显示所有本地分支，当前所在分支前会有 `*` 标记。

---

### 总结表格

| 操作       | 命令示例                  |
| ---------- | ------------------------- |
| 创建分支   | `git branch feature-x`    |
| 切换分支   | `git switch feature-x`    |
| 创建并切换 | `git switch -c feature-x` |
| 合并分支   | `git merge feature-x`     |
| 变基分支   | `git rebase main`         |
| 查看分支   | `git branch`              |
| 删除分支   | `git branch -d feature-x` |
